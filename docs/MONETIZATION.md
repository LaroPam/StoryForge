# Monetization Strategy

## Principle

The product should not sell access to a model.

It should sell:

- personal adventure
- emotional immersion
- story continuity
- beautiful illustrated scenes
- longer campaigns
- memory
- exportable personal books
- co-op experiences
- creator worlds

## Recommended monetization ladder

```txt
Free → Premium subscription → Credits → Illustrated export → Creator marketplace
```

## MVP rule

The MVP must be monetization-ready, but payments must not be implemented in the first phases.

Do not add billing, checkout, subscriptions, webhooks, or payment providers until explicitly requested.

## Free tier

Possible limits:

- limited daily turns
- limited active stories
- basic memory
- limited illustrations
- short adventures only
- lower priority generation later

Goal:

Let the user experience the magic before asking for money.

## Premium subscription

Possible benefits:

- more turns
- longer campaigns
- extended memory
- more active stories
- premium visual styles
- key scene illustrations
- story library
- PDF export
- duo mode

## Credits

Credits can be used for:

- scene illustrations
- image regeneration
- premium model turns
- cover generation
- voice narration
- export features
- longer scenes
- special endings

Credits are important because AI image generation and heavy generation can create variable costs.

## One-time purchases

Possible purchases:

- illustrated story export
- poster generation
- gift story
- premium world packs
- special visual style
- campaign continuation

## Creator marketplace

Later-stage model:

- users create worlds
- users publish worlds
- paid worlds
- creator subscription
- platform commission
- future revenue sharing

This should not be part of MVP.

## First paid conversion moments

Best moments to show paid offer:

1. User reaches free turn limit and wants to continue.
2. User wants a longer campaign.
3. User wants illustrated scenes.
4. User finishes a story and wants a PDF/book export.
5. User wants to invite a second player.
6. User wants to create a custom world.

## Monetization-aware domain fields

Future domain model should be able to support:

- user plan
- credit balance
- daily turn limit
- image credit limit
- active story limit
- premium visual styles
- can export book
- can use long campaigns
- can use duo mode
- can create worlds

Do not hardcode unlimited usage.
